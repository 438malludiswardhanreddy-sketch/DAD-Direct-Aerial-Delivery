# DAD Engineering Logbook

This directory contains the chronological engineering logs for the Direct Aerial Delivery (DAD) project bring-up and physical validation phases. It tracks daily objectives, system changes, issues identified, and evidence collected.

## Log Entry Template

When creating a new log entry (e.g., `YYYY-MM-DD.md`), use the following markdown template:

```markdown
# Engineering Log: YYYY-MM-DD
**Author**: [Your Name]
**Milestone Target**: [e.g., v0.4.0-ground-validation]

---

### 1. Objective
*What was the goal of today's work session?*

### 2. Work Performed
*Detail the step-by-step assembly, calibration, testing, or code changes.*

### 3. Issues Found
*What went wrong? List any error codes, connection dropouts, physical constraints, etc.*

### 4. Fixes Applied
*How were the issues resolved? (e.g. parameter adjustments, wiring corrections, software patches).*

### 5. Evidence Collected
*Link to the trace assets collected during the session:*
*   **Photo**: `[Link to photo](file://...)`
*   **Video**: `[Link to video](file://...)`
*   **CSV/Log**: `[Link to telemetry file](file://...)`

### 6. Next Actions
*What needs to be accomplished in the next session?*
```

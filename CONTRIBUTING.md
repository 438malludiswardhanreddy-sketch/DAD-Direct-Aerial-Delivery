# Contributing to DAD (Direct Aerial Delivery)

First off, thank you for considering contributing to DAD! It's people like you that make DAD such a robust autonomous drone delivery platform.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct:
* Use welcoming and inclusive language.
* Be respectful of differing viewpoints and experiences.
* Gracefully accept constructive criticism.
* Focus on what is best for the community.
* Show empathy towards other community members.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* Open a new issue with a clear title, detailed description, steps to reproduce, and environment details (OS, Python version, PX4/ArduPilot setup).

### Suggesting Enhancements
* Open an issue explaining the proposed feature and why it would be beneficial.
* Provide architectural diagrams or code snippets if possible.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. Install testing dependencies and run the test suite.
3. Keep code coverage high and follow formatting conventions:
   * **C++**: Google C++ Style Guide.
   * **Python**: PEP 8 style guide.
4. Ensure your commits are descriptive and signed off if possible.
5. Submit a pull request and describe your changes in detail.

## Questions?
Feel free to open an issue or reach out to the core maintainers.

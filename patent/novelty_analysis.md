# Novelty and Patentability Analysis
This document evaluates the novelty and inventiveness of the DAD system architecture, identifying the patentable claims.

---

## 1. Novel Aspects of the Architecture
The primary novelty of the DAD system lies in its low-cost, resource-constrained implementation of autonomous perception and failsafe routing:

### 1.1. Asymmetric Sensor Alignment and EKF projection
Traditional systems use homogeneous 3D point clouds (from 3D LiDARs) for spatial mapping. DAD uses an **asymmetric sensor configuration**:
*   A single-beam (1D) LiDAR.
*   A single monocular 2D camera.
*   *Novel Step*: The algorithm projects 1D distance measurements into the 2D bounding box generated by the camera, and uses a Kalman Filter Jacobian matrix to reconstruct 3D tracking states ($p_x, p_y, p_z, v_x, v_y, v_z$). This achieves 3D tracking capabilities using components that cost less than ₹7,000 (USD $80.00).

### 1.2. Dynamically Constrained Emergency Diversion (Safe Havens)
Rather than executing a simple Return-to-Launch (RTL) to the starting station (which is unsafe under severe battery sag or sudden monsoon rain), the DAD navigation engine maps safe landing centres:
*   *Novel Step*: Real-time environmental sensor data (precipitation rate, wind speed) is fused with current battery discharge profiles. If thresholds are breached, the system executes a dynamic voronoi partition to steer the drone to the closest Emergency Landing Pad.

## 2. Inventiveness Analysis (Indian Patent Act Compliance)
Under **Section 2(1)(ja)** of the Indian Patent Act 1970, an invention must possess an "inventive step" that is not obvious to a person skilled in the art.

*   **Technical Advancement**: The combination of single-beam rangefinders with 2D object detection models to solve 3D tracking represents a clear technical advancement in cost and payload reduction.
*   **Economic Significance**: Enables low-weight delivery quadrotors to meet DGCA safety norms without exceeding the 25kg Small Category MTOW limit.
*   **Non-Obviousness**: Fusing 1D range with 2D bounding boxes is non-obvious, as standard practices rely on stereoscopic camera setups or multi-beam LiDAR arrays.
